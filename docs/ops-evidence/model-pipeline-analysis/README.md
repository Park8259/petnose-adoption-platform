- image 12

Worst-case improvement:

- image 13
- 0.7556 -> 0.7952
- delta +0.0396

Interpretation guardrails:

- This is not classification accuracy improvement evidence.
- Use these terms: same-dog matching stability improvement, worst-case similarity improvement, reference coverage improvement, and robustness improvement against capture variation.
- Multi-reference heatmap values are target-level max aggregation over Refs 1-5. They are not centroid scores.

## Local PyTorch Batch Inference Benchmark

Scope: local CPU controlled experiment.

5 separate embeds:

- mean: 2532.397 ms
- P95: 2781.707 ms

`/embed-batch` with 5 images:

- mean: 1692.285 ms
- P95: 1836.452 ms

Result:

- mean reduction: 33.17%
- P95 reduction: 33.98%
- mean time saved: approximately 840.112 ms
- speedup: approximately 1.496x

Mean is average processing time. P95 means 95% of observed runs completed within that time. This was the structural improvement adopted in the runtime path: call/forward structure was improved without model retraining or threshold changes.

See [local_batch_inference_summary.md](local_batch_inference_summary.md).

## Local ONNX Runtime Model-Only Benchmark

Scope: local model-only preliminary experiment.

Single image:

- PyTorch mean: 489.619 ms
- PyTorch P95: 548.287 ms
- ONNX Runtime mean: 116.898 ms
- ONNX Runtime P95: 141.394 ms

Batch of 5:

- PyTorch mean: 1630.565 ms
- PyTorch P95: 1754.602 ms
- ONNX Runtime mean: 507.514 ms
- ONNX Runtime P95: 566.716 ms

Result:

- batch mean speedup: approximately 3.21x
- batch P95 speedup: approximately 3.10x

Vector parity:

- cosine similarity minimum: 1.00000000
- max absolute difference maximum: 9.87e-8
- L2 difference maximum: 1.11e-6

These values exclude FastAPI, HTTP, Docker, Spring-Python network, and AWS overhead. Do not state that the production API became 3.21x faster. This is a follow-up validation candidate, not an adopted production change.

See [local_onnx_runtime_summary.md](local_onnx_runtime_summary.md).

## Local Runtime Tuning Reviewed But Not Adopted

- warm-up: no evidence of first-request improvement that met the adoption bar
- channels_last: no meaningful latency improvement
- warm-up + channels_last: below adoption criteria
- torch.compile: unavailable in the tested Python 3.12 + torch 2.3.1 combination

No runtime tuning flags were applied to product code.

See [local_runtime_tuning_summary.md](local_runtime_tuning_summary.md).

## Reproducibility

The reproducibility script is [../../model-analysis/run_ddubi_single_vs_multi_heatmap.py](../../model-analysis/run_ddubi_single_vs_multi_heatmap.py).

Example:

```powershell
python docs/model-analysis/run_ddubi_single_vs_multi_heatmap.py `
  --image-dir <fixture-dir> `
  --output-dir <evidence-dir> `
  --reference-images 1-5 `
  --target-images 6-13 `
  --model-dir <model-dir> `
  --device cpu
```

The script imports `python-embed/app/embedding/dog_nose_identification2_embedder.py` from the repository checkout. It reads raw images only from `--image-dir`, writes only PNG/CSV/markdown summaries, and does not write raw images, raw vectors, checkpoint contents, or private absolute paths.
